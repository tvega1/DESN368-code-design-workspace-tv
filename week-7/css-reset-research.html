CSS Reset, Normalize.css, & Boilerplates — Research

Class: DESN 368 · Week 7

1. Why reset at all?

Browsers all come with their own default CSS — different margins, fonts, spacing, button styles, etc. These inconsistencies cause layout problems. A CSS reset removes these default styles so every browser starts from the same baseline.

2. Eric Meyer (2007) vs Josh Comeau (Modern Reset)

Eric Meyer Reset (2007)

  • Extremely aggressive — removes ALL margins, padding, borders.
  • Forces you to restyle everything manually.
  • Goal: total control over your design.

Josh Comeau’s Modern Reset

  • Keeps useful defaults (less destructive).
  • Adds modern best practices like box-sizing: border-box;.
  • Improves text rendering and scrolling.

Why Josh over Eric?

  1. Eric’s reset wipes too much — lots of extra work.
  2. Josh’s reset uses modern features and accessibility improvements.
  3. Josh keeps helpful defaults so you restyle less.

3. How to use a reset or normalize

You include it at the TOP of your CSS like this:

<link rel="stylesheet" href="reset.css">
<link rel="stylesheet" href="styles.css">

4. What’s a CSS boilerplate?

A boilerplate is MORE than a reset. It includes:

  • Reset or normalize styles
  • Base typography
  • Starter layout styles

Basically: a starter kit for CSS projects.

5. Reset vs Normalize.css

Reset

  • Removes EVERYTHING.
  • You rebuild all styles from zero.
  • Keeps useful defaults.
  • Makes browsers consistent.
  • Smoother and more user-friendly.
  • Less extreme.
